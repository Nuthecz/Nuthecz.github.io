Lab7

理解

​ 这里就是要实现一个 alias-aware 的过程间常量传播分析。我们知道常量传播在 Lab2 喝 Lab4 中都出现过，这里就是分析新的情况，在别名情况下的过程间指针分析。

题目相关

  • 为 Java 实现一个 alias-aware 的过程间常量传播分析

Alias-Aware 常量传播

analysis

  • pascal.taie.analysis.pta.PointerAnalysisResult
    • 该类提供了一系列查询指针分析结果的 API，提供有上下文敏感喝非敏感的 API，由于过程间常量传播不需要知道指针分析结果中的上下文信息，所以应该使用不带上下文的指针分析结果

ir

  • pascal.taie.ir.exp.ArrayAccess
    • 该类代表了数组访问表达式，例如 a[i]，该类的实例在类 StoreArray 和 LoadArray 的实例中出现。

具体代码

代码总览

​ 需要完成以下的类，分别为InterConstantPropagation的6个API和InterSolver的2个API

  • boolean transferCallNode(Stmt,CPFact,CPFact)
  • boolean transferNonCallNode(Stmt,CPFact,CPFact)
  • CPFact transferNormalEdge(NormalEdge,CPFact)
  • CPFact transferCallToReturnEdge(CallToReturnEdge,CPFact)
  • CPFact transferCallEdge(LocalEdge,CPFact)
  • CPFact transferReturnEdge(LocalEdge,CPFact)
  • void initialize()
  • void doSolve()

initialize

​ 这里主要是添加 allVarAliases，staticFieldStore 和 staticFieldLoad 三个数组，前者通过遍历所有的 var，查看不同指针的指针集是否存在交集而构建别名信息的表 Map<Var, HashSet<Var>>，后两者则是由于是静态字段，它没有别名，所以直接遍历 icfg 添加相应的 store 和 load 语句即可，以便后续的使用。

isArrayIndexAlias

​ 这里就是对于数组 a,b 的指针集有交集的情况下，根据常量传播中的 i 和 j 的结果来决定 a[i] 和 b[i] 是否为别名。对应的就是下面的表格。

a[i] and b[j] j = UNDEF j = c2 j = NAC
i = UNDEF 不互为别名 不互为别名 不互为别名
i = c1 不互为别名 当且仅当 c1=c2 时互为别名 互为别名
i = NAC 不互为别名 互为别名 互为别名

transferCallNode

​ 这里就是增加对于别名信息的处理，大体上分为 FieldStore，FieldLoad，ArrayStore，ArrayLoad 四种处理，其中前两者存在实例字段和静态字段的区别。对于 load 和 store 的处理就是下面的规则。

  • 对于 load，把 对应 的 store 的 rhs 用 meet 计算出来，赋给 load 的 lhs
  • 对于 store，若 y.f 的值有变化，把 对应 的 load 加入 worklist

​ 我自己做的时候需要注意以下两点：

  • 根据手册中的常量化处理 canHoldInt 来进行操作，但是这只是对于 load 相关语句而言，对于 store 相关语句不需要这个，即不需要判断 a.f 是否为一个常量，如果对于 store 也判断会跳过某些需要处理的样例
  • 对于 Store 语句的返回需要直接 return。这里我依然不是很明白，上面写的进入的 cp.transferNode 的分支不同，也确实是这样，但是若是对于一个 store 语句而言，没有立即返回而是等到最后才返回的效果应该一样吧，毕竟都是同样的语句。但是这里任意一个的删除都会产生相应的影响，目前的想法是这里如果没有 return isChange，那么就经过了两次 cp.transferNode，第一次确实是处理 store 语句，但是处理后会改变 out 的值，所以第一次和第二次的结果实际上是不同的，因此应该返回第一次处理的结果。

addWorkList

​ interSolver中添加的，主要服务于上面 store 添加到 worklist 的处理。之后其余的地方都没有改动，直接复制前面 Lab 的代码即可。
