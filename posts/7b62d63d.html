污点分析

plugin

  • pascal.taie.analysis.pta.plugin.taint.TaintTransfer

    • 这个 record 类表示污点传播。在这个类中，我们用整数来表示污点传播被对应方法引发时的 from 变量和 to 变量。具体来说，一个大于等于 0 的整数 i 表示调用点上被调用方法的第 i 个参数；-1 表示 base 变量；-2 表示接收结果的变量。这里就是对于 taintTransfer.from() 和 taintTransfer.to() 两个 Int 类型，他们使用整数来表示一个方法调用的各个部分。

  • pascal.taie.analysis.pta.plugin.taint.TaintConfig

    • 这个类表示污点分析的配置信息。它提供了解析配置文件以及获取 sources、sinks 和污点传播信息的 API。它读取taint-config.yml 配置的source，sink，transfers信息，然后利用 api 传递出去。

  • pascal.taie.analysis.pta.plugin.taint.TaintManager

    • 这个类被用来管理污点分析中的污点对象，通过 makeTaint 方法构建污点对象 obj，之后使用指针分析的 getCSObj 给一个空上下文，构建 csObj。

  • pascal.taie.analysis.pta.plugin.taint.TaintFlow

    • 这个 record 类表示分析算法检测到的 taint flows（由 source 的调用点和 sink 的调用点组成），也就是污点分析算法的结果。它不需要参与 PTA 的过程，因此可以根据最后的 pts 来处理污点流，所以这里通过对于所有的 Sink csCallSite 进行遍历，构建最后的 <j,l,i> -> taintFlows

代码总览

​ 需要补全的 pascal.taie.analysis.pta.cs.Solver 的五个方法

  • void addReachable(CSMethod)
  • void addPFGEdge(Pointer,Pointer)
  • void analyze()
  • PointsToSet propagate(Pointer,PointsToSet)
  • void processCall(CSVar,CSObj)

​ 同时补全pascal.taie.analysis.pta.plugin.taint.TaintAnalysiss 中的一个方法，添加适当的API进行构造

  • Set<TaintFlow> collectTaintFlows()

analyze

​ 这里就是在 PointsToSet delta = propagate(n, pts); 下面添加语句，taintAnalysis.propagate(n, delta);，这个语句就是处理 delta 中的污点对象，在 taintAnalysis 中实现一个 propagate 方法，获取 delta 中的污点对象，然后将其后继节点加入 WorkList 进行处理，从而将这个污点进行传播

processCall

​ 这里修改了 A6 中的静态调用和动态调用的共存部分，通过将中间部分进行扩展获得更多的信息，从而提供给污点传播进行利用，但是其效果和之前实现的并没有区别，这里主要在处理的中间方法 processEachCall 添加对于污点的 Source，Sink，TaintTransfer 的处理，这里就是因为它们只会发生在 Invoke 中，所以直接在这个必经之处进行添加处理即可

​ 这里在没有创建 edge 的条件下进行污点传播的分析，通过添加不同的语句分别对于上面三者进行处理。通过在taintAnalysis 中实现 dealSource，dealSinkCallSite，dealTaintTransfer 三种方法进行处理。

​ 对于 Source，就是查看方法调用的 method 是否为给定的 Source 中的一个，若是，则添加一个指针的执行，使调用方法的左值(返回值)指向一个污点对象集

​ 对于 Sink，则是判定 method 是否为配置中的 Sink，若是，则添加到 sinkCallSites 中，保存了污点分析所有的 sink 调用语句，以便后续在 collectTaintFlows 中形成 TaintFlows。

​ 这里是 TaintTransfer 的处理，对于三种传播规则，base-to-result，Arg-to-base，Arg-to-result进行处理，然后通过构建 addTFGEdge 添加污点边，从而给上文的 delta 的传播提供传播的污点传播流，进行污点信息的传递。

collectTaintFlows

​ 这里就是对于最后的 TaintFlow 进行处理，通过遍历所有的 Sink csCallSite 来获得最后的 TaintFlow流

辅助设置

​ 这里仿照 PFG 构造了 TFG，然后在这个污点传播图中进行污点的传播处理，也就是对应 TaintTransfer。然后这里构建了 SinkCallSite，提供给最后 TaintFlow 的构造。
